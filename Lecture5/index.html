Ручное тестирование интерфейсов

Исследовательское

Свободный поиск неочевидных дефектов, опора на опыт и интуицию. Вопросы: «А что, если дважды нажать?», «А если ввести буквы в поле для телефона?»

По тест-кейсам

Чёткие шаги, предусловия, ожидаемые результаты. Пример тест-кейса «Добавление товара в корзину».

Предусловия: пользователь авторизован, товар в каталоге.

Шаги: 1. Нажать «В корзину» у товара «Смартфон». 2. Нажать иконку корзины.

ОР: В корзине отображается товар с правильной ценой.

F12 — лучший друг тестировщика

Инструменты разработчика в браузере

Elements

HTML/CSS структура. Проверка вёрстки, стилей, классов. Точное описание багов.

Console

Логи JavaScript, ошибки. "Uncaught TypeError" — сразу видно проблему.

Network

Все запросы, статусы, время, тело ответа. Связь UI и API.

Application

LocalStorage, куки, сессии. Очистка данных для тестов первого входа.

Performance

Анализ скорости загрузки, узкие места.

Lighthouse

Аудит производительности, доступности, best practices.

Инструменты разработчика превращают тестировщика в детектива: от «кнопка не работает» к «почему не работает».

Первый шаг в автоматизацию

Selenium WebDriver

Библиотека для управления браузером из кода (C#, Java, Python, JS). Драйверы: ChromeDriver, GeckoDriver, EdgeDriver.

Простой тест логина (C#) 
using OpenQA.Selenium;
using OpenQA.Selenium.Chrome;

IWebDriver driver = new ChromeDriver();
driver.Navigate().GoToUrl("https://localhost:5001/login");

IWebElement emailField = driver.FindElement(By.Id("email"));
emailField.SendKeys("test@example.com");

IWebElement passwordField = driver.FindElement(By.Name("password"));
passwordField.SendKeys("TestPassword123");

IWebElement loginButton = driver.FindElement(By.CssSelector("button[type='submit']"));
loginButton.Click();

System.Threading.Thread.Sleep(2000);
IWebElement header = driver.FindElement(By.TagName("h1"));

if (header.Text.Contains("Личный кабинет"))
    Console.WriteLine("Тест пройден");
else
    Console.WriteLine("Тест провален");

driver.Quit();

Проблемы "сырого" Selenium:

  • Хрупкость (изменился ID — тест упал)
  • Читаемость (копипаста)
  • Поддерживаемость (править все тесты при изменении страницы)
Решение проблем

Паттерн PageObject

Для каждой страницы или значимого блока — отдельный класс, инкапсулирующий элементы и действия.

LoginPage

public class LoginPage
{
    private IWebDriver driver;
    private By emailField = By.Id("email");
    private By passwordField = By.Name("password");
    private By loginButton = By.CssSelector("button[type='submit']");

    public LoginPage(IWebDriver driver) { this.driver = driver; }

    public void EnterEmail(string email) =>
        driver.FindElement(emailField).SendKeys(email);

    public void EnterPassword(string password) =>
        driver.FindElement(passwordField).SendKeys(password);

    public void ClickLogin() =>
        driver.FindElement(loginButton).Click();

    public DashboardPage LoginAs(string email, string password)
    {
        EnterEmail(email);
        EnterPassword(password);
        ClickLogin();
        return new DashboardPage(driver);
    }
}

DashboardPage

public class DashboardPage
{
    private IWebDriver driver;
    private By header = By.TagName("h1");

    public DashboardPage(IWebDriver driver)
    {
        this.driver = driver;
    }

    public string GetHeaderText() =>
        driver.FindElement(header).Text;
}

Тест с PageObject

IWebDriver driver = new ChromeDriver();
driver.Navigate().GoToUrl("https://localhost:5001/login");
var loginPage = new LoginPage(driver);
var dashboardPage = loginPage.LoginAs("test@example.com", "TestPassword123");
Assert.IsTrue(dashboardPage.GetHeaderText().Contains("Личный кабинет"));
driver.Quit();

Читаемость, переиспользование, лёгкость поддержки.

Что автоматизировать?

Стратегия: Smoke, регресс, CI/CD

  • Smoke-тесты — критический функционал (логин, оплата)
  • Регресс — повторяющиеся проверки
  • CI/CD — запуск автотестов при каждом коммите (Jenkins, GitHub Actions)
  • Headless-режим — без графического интерфейса, для серверов
  • Selenium Grid — параллельный запуск в разных браузерах

Автоматизация имеет стоимость написания и поддержки. Нестабильные тесты вредят больше, чем помогают.

UI и безопасность

Автоматизация поиска уязвимостей

XSS-инъекции

Автотест вводит <script>alert('XSS')</script> во все поля и проверяет выполнение.

Brute force

Тест проверяет блокировку после N неудачных попыток входа.

Back after logout

Проверка, что кнопка «Назад» не возвращает в личный кабинет после выхода.
